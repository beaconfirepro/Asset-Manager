ITM (Inspection, Testing & Maintenance) — Layout Wireframes

Structural zone reference. Companion to itm_UI_SPEC.md
Grid: 4 columns (Main Nav, Left Panel, Content, Right Panel) x 6 rows.
[PSB] = Page Specific Buttons, [PF] = Page Filters — defined below each layout.
Data marked HubSpot is read via API (reference + cache); ITM owns inspection execution & compliance.

KEY:   Set globally    Hidden on toggle click

14. INSPECTION WORKSPACE

Route: /itm/inspections/:id

Context: inspectionId (keyed to the HubSpot Inspection ticket ID, from route param :id). Offline-first PWA full-screen form for field crews. Loads the assigned inspection (asset, form schema, prior answers) into local storage; syncs on reconnect. A deficient answer creates a HubSpot Deficiency ticket via API on sync. No app-shell grid — mobile full-screen layout.
MOBILE FULL-SCREEN FORM
Top bar: [Back] | Asset / Inspection title | C-22 OfflineSyncIndicator (online / offline / pending-sync)
Question Set
C-18 QuestionSetRenderer: renders form-schema questions (pass/fail/NA, numeric, text), progress indicator, conditional logic.
Deficiency Capture
C-19 DeficiencyCaptureCard: log deficiency per failed question (severity low/high/critical, note, location, link photo). On submit, creates a HubSpot Deficiency ticket via API.
Photo Annotate
C-20 PhotoAnnotator: capture/attach photo, draw/annotate, caption; attaches to question or deficiency.
Signature
C-21 SignaturePad: technician + on-site contact signatures; name/date capture.
Offline Sync
C-22 OfflineSyncIndicator: connection state, queued-changes count, [Sync Now]; auto-syncs on reconnect.
Bottom action bar: [Save Draft (local)] | [Submit for QA] (queues if offline)
Layout: Mobile full-screen form (PWA, dark-mode-friendly per Field Operations conventions). No Main Nav / LP / RP / app-shell grid.
Sections (sub-tables): Question Set (C-18), Deficiency Capture (C-19), Photo Annotate (C-20), Signature (C-21), Offline Sync (C-22)
Offline behavior: Loads to local storage; all edits persist locally; submit queues when offline and flushes on reconnect.
